# Промт-специфікація: додаток-компаньйон «Медовик»

> Готовий промт / навчальний матеріал. Можна віддати AI-асистенту або розробнику для відтворення додатка. Продукт: рецепти крему + збірка торта з готових коржів.

---

## 0. МЕТА ТА ФІЛОСОФІЯ

Виробник продає **напівфабрикат** (готові коржі), а покупець залишається з питанням «що з ними робити». Додаток знімає це тертя: дає **цінність до контакту** — рецепти й покрокову збірку — ще до й після покупки.

Принцип: продає не сам продукт, а **подача + інструмент**. Додаток = подарунок, а не форма замовлення.

Формула сторінки: **візуальний якір → короткий текст → покроковий інструмент**.

---

## 1. ТЕХНІЧНІ РАМКИ

- **Один файл `index.html`** — HTML + CSS + JS + зображення (base64) всередині. Зовнішнє — лише Google Fonts.
- **Без бекенду, без збору персональних даних.**
- **Тримовність UA / EN / NL** через об'єкт `I18N` + перемикач; уся статика й рецепти перекладені; одиниці виміру мають словник перекладу (г→g, ст.л→tbsp/el тощо).
- **OG-теги** (`og:title`, `og:description`, `og:image`, `og:url`) для прев'ю-картки при поділі лінка.
- **Розгортання:** статичний хостинг; файл має називатися `index.html` у корені.

---

## 2. РОЗДІЛИ СТОРІНКИ (згори вниз)

### 2.1. Шапка (hero)
- **Перемикач мов** UA / EN / NL — фіксований угорі праворуч, приглушена плашка. Тап перекладає весь додаток.
- **Eyebrow** — дрібний рядок-категорія («домашній медовик»).
- **Заголовок** — назва продукту, серифним/характерним шрифтом.
- **CSS-ілюстрація-підпис** — фірмовий знак: розріз торта з чергуванням шарів «корж / крем», який анімовано «збирається» при завантаженні (шари з'являються по черзі). Стрічки крему перефарбовуються під обраний рецепт.
- **Підпис під ілюстрацією** — коротка формула (напр. «крем в холодильнику · 2 год, краще на ніч»). Без прив'язки до конкретної кількості коржів (щоб не суперечити перемикачу 5/10).
- **Note-блок** — пояснення «у вас коржі, а не готовий торт — і це добре: зберете свіжий, під себе».

### 2.2. Розділ «Крем» (01)
- **Перемикач рецептів** — кнопки, що переносяться в кілька рядків. Приклад набору з 5:
  1. Класичний (сметанний, канонічний рецеп�т, жирність вказана явно);
  2. З вершками (адаптація під низькожирну місцеву сметану);
  3. Від шефа (фруктовий джем + сезонні варіації);
  4. Вишневий (кислинка; окрема **доросла порада** про алкоголь, виділена червоним);
  5. Несолодкий (фруктове пюре замість цукру; **чесна засторога** про алергени — мед/глютен у коржах не прибираються кремом).
- **Кожен рецепт має власний акцентний колір** — при перемиканні перефарбовується акцент інтерфейсу і стрічки крему в ілюстрації.
- **Список інгредієнтів** — із **живим перерахунком** під кількість коржів.
- **Лічильник кількості** (напр. 5 / 10) — множить усі інгредієнти; об'єми пораховані під реальний торт (діаметр, прошарки, верх, боки).
- **Підказка-tip** під складом — метод + застереження. Підтримує вкладені блоки: сезонні варіації (звичайний колір) і дорослі поради (червоний).

### 2.3. Розділ «Збірка» (02)
- **Фото/ілюстрація** процесу (реальне фото робіт краще за рендер).
- **Кроки з галочками** — тап відмічає крок (зручно, коли руки в креме). Приклад:
  1. Корж → шар крему → корж, до останнього;
  2. За бажанням обмазати верх/боки, обсипати крихтою;
  3. Холодильник — час витримки;
  4. Декор на смак (фрукти, ягоди, крихта, м'ята/базилік);
  5. Подавати охолодженим.

### 2.4. Футер
- **Кнопка «Замовити коржі»** — веде на інстаграм/контакт **виробника-партнера** (замовлення й оплата — його).
- **Підпис виробника** (ім'я + хендл).
- **Тихий підпис розробника** — приглушений, клікабельний на WhatsApp розробника (лід-канал: інші виробники бачать і можуть звернутися).

## 4. UX / UI — ПРАВИЛА

- **Мобільний-first**, колонка до ~520 px, центрована.
- **Палітра — тепла, «медова»:** папір/крем, бурштин-золото як акцент. Кожен рецепт — власний відтінок акценту (золото, мідь, зелень, вишня, нейтраль).
- **Шрифти:** заголовки — тепла серифна (Literata); тіло — чистий гротеск (Manrope). Обов'язкова кирилиця + латиниця для трьох мов.
- **CSS-ілюстрація торта** — фірмовий знак, не картинка; анімація складання шарів; перефарбування під рецепт.
- **Живий перерахунок** інгредієнтів — головна інтерактивна цінність.
- **Чесність у копірайтингу:** реальні об'єми, канонічні рецепти, **прямі застереження про алергени** замість заспокійливих неправд. Дорослі поради (алкоголь) — явно марковані й виділені кольором.
- **Без обіцянок-гарантій.** Формулювання нейтральні й точні.

---

## 5. КОНТЕНТ-ФОРМУЛА ЗАПУСКУ (супровід продукту)

- **Пост-хук:** зображення-хук → короткий текст → лінк на додаток. Один вхід (особисті повідомлення).
- **Прев'ю-картка лінка** через OG-теги (фото + заголовок).
- **Чесний лік проданого** («- N») без накрутки — соціальний доказ.
- **Воронка постів:** нагадування → «настрій» → чесне закриття (реальний дедлайн) → підсумок/подяка.

## 6. ЧЕК-ЛИСТ ПЕРЕД ЗАПУСКОМ

- [ ] Файл — `index.html`, у корені папки, що деплоїться.
- [ ] Заглушки замінені (WhatsApp, інстаграм, імена, домен).
- [ ] Три мови повні; одиниці виміру перекладаються.
- [ ] Об'єми інгредієнтів пораховані під реальний торт (5 і 10).
- [ ] Застереження про алергени присутні; дорослі поради марковані.
- [ ] OG-теги з реальним доменом; прев'ю-картка перевірена.
- [ ] Перевірено на реальному телефоні.

## 7. ГОЛОВНЕ ПРАВИЛО

Продукт можна вдосконалювати нескінченно (шрифт, колір, ілюстрація). Але найважливіший крок — **перший продаж живій людині**. Додаток — інструмент і тест, а не самоціль.
